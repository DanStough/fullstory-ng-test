Instructions

Welcome to a short demo of FullStory! This site demostrates how you can leverage the FullStory (FS) and Github REST APIs to create an interactive support ticketing system with references to live session recordings.

Following the instructions below will lead you on a tour of this demo's functionality.

Warnings, Notes & Assumptions

Before we begin a few words of caution:

  • This site was not designed with responsiveness in mind; if you're on mobile, you're going to have a bad time.
  • This is no session storage; this means if you refresh the page, all of the changes you have been made will be reset.
  • If FullStory recording isn't working, check that your Ad Blocker isn't blocking FullStory. You could spend more time trying to "debug" this then you would like ;) .

Trying out the App

1. Configure

In the "Configure" tab, you can change the FS client authentication; this can let you generate multiple faux sessions/users. Setting this configuration is optional.

2. Record

With the configuration items in place, you can start a FullStory session using the ???Record button in the menu???. The "Playground" tab is provided as a place to experiment with Rage Clicks, Abandoned forms and excluded areas.

Click the button again (now a Stop Sign) to ????end the session????.

3. Submit a Support Ticket

Once you have some session recordings, you can browse them under the "Support" tab. There may be some delay in the recording capture.

When you identify a "problematic" session where you had an issue, click on the "Submit Ticket" button for that issue. Fill the details into the dialog a Github issue with the session info will be created in the Github project.
