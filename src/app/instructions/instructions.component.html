Instructions

Welcome to a short demo of FullStory! This Angular site demostrates how you can leverage the FullStory and Github REST APIs to create an interactive support ticketing system with references to live session recordings using only the front end of a web application. Following the instructions below will lead you on a tour of this demos functionality.

Warnings, Notes & Assumptions

Before we begin a few words of caution:

  • This site was not designed with responsiveness in mind; if you're on mobile, you're going to have a bad time.
  • This is a front-end-only application that currently has no session storage; this means if you refresh the page, all your confguration will be wiped.
  • If FullStory recording isn't working, check your Ad Blocker isn't blocking FullStory. You could spend more time trying to "debug" this out then you would like.

Configure

In the "Configure" tab, setup the follow credentials (hopefully you have them handy):

  1. FullStory Account Information - for recording.
  2. FullStory API Key (Pro Subscription Only) - for viewing sessions over the rest API.
  3. Github API Key - for creating issues against the project.

Record

With the configuration items in place, you can start a FullStory session using the ???Record button in the menu???. The "Playground" tab is provided as a place to experiment with Rage Clicks, Abandoned forms and excluded areas.

Click the button again (now a Stop Sign) to ????end the session????.

Submit a Support Ticket

Once you have some session recordings, you can browse them under the "Support" tab. There may be some delay in the recording capture.

When you identify a "problematic" session where you had an issue, click on the "Submit Ticket" button for that issue. Fill the details into the dialog a Github issue with the session info will be created in the Github project.
